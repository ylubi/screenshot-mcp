- `继续`
- `暂时先完成 windows 吧`

- `Error connecting to MCP server: MCP error -32601: Method not found: initialize`
- `到这里就停止了，没有任何输出`

- `应该支持进程名进行查找`

- `我说的是 ai 调用 mcp 时的描写信息，而不是简单的文档`

- `docs\PROMPTS_USED.md 最开始的部分不完全，应该有更早的信息`

## 有效提示词技巧

### ✅ 推荐做法

1. **直接粘贴错误信息**
   - 完整的错误堆栈
   - 具体的错误代码

2. **明确具体需求**
   - 说明想要什么功能
   - 指出缺少什么

3. **描述问题现象**
   - 说明发生了什么
   - 预期应该怎样

4. **及时纠正方向**
   - 发现理解偏差立即指出
   - 明确真正的需求

5. **澄清误解**
   - 区分文档和接口描述
   - 说明具体指的是什么

### ❌ 避免做法

1. **过于模糊** - "帮我改一下"
2. **缺少上下文** - "这个不对"
3. **一次要求太多** - 同时要求多个不相关功能

2. **及时反馈** - 发现问题立即报告
3. **明确需求** - 清楚表达想要的功能

## 项目规范

项目使用 Kiro Steering 功能，包含以下规范文件：

- `product.md` - 产品概述
- `tech.md` - 技术栈
- `structure.md` - 项目结构
- `项目规范.md` - 编码规范

这些规范确保：
- 代码风格一致
- 中文文档，英文代码
- 单文件不超过 500 行
- 清晰的目录结构
